Records only what is **not** in the plan: deviations, decisions where the spec was silent, and things a reviewer should know.

## Versions / framework API (confirmed against installed packages)

- `@tanstack/react-start@1.168.25`, `@tanstack/react-router@1.170.15`, Vite 8, `@vitejs/plugin-react@6`.
- Vite plugin: `tanstackStart` from `@tanstack/react-start/plugin/vite`, placed **before** `viteReact()`.
- SPA mode option shape used: `tanstackStart({ spa: { enabled: true }, pages: [{ path }] })` (`pages` is top-level, not nested under `spa`). `spa.prerender` has schema defaults.
- No `src/client.tsx` / `src/server.ts` entry files needed — the plugin supplies defaults. Only `src/router.tsx` (`getRouter()` → `createRouter({ routeTree, scrollRestoration: true })`) and `src/routes/__root.tsx` were authored. `routeTree.gen.ts` is generated by the plugin and gitignored.

## Deviation: `"type": "module"` on apps/web

The TanStack Start Vite plugin is ESM-only; loading `vite.config.ts` failed under CJS. Added `"type": "module"` to `apps/web/package.json`.

## Deviation: home keeps `ClientProviders` (plan said remove it)

The plan assumed home's marketing sections needed no providers and removed `ClientProviders` to prerender non-empty HTML. **Wrong** — `src/components/home/hero-section.tsx` calls `useAccount()` and `useAudioPlayer()`, so prerendering home without providers threw `useAccount must be used within an AccountProvider`. Reverted: home keeps `ClientProviders` and stays client-rendered (exactly as under Next — home rendered `null` server-side there too). Result: `/` is served by the SPA shell `_shell.html` (full global head: title, OG/Twitter, favicons, Umami w/ `crossorigin`, fonts) and hydrates client-side. The **legal pages (`/privacy-policy`, `/terms-of-service`) prerender with real content** for SEO; they use no providers. `/dashboard` is also crawled+prerendered (gated → empty shell, harmless).

## Deviation: cross-origin isolation via middleware plugin (not `server.headers`)

`server.headers` / `preview.headers` do **not** reach TanStack Start's SSR/document response, so the top-level HTML at `/` was served without COOP/COEP — i.e. not cross-origin isolated, which breaks WebLLM's `SharedArrayBuffer` locally. Replaced them with a small Vite plugin (`crossOriginIsolationHeaders` in `vite.config.ts`) that sets the three headers in `configureServer`/`configurePreviewServer` middleware on every response (including the document). Production headers come from `vercel.json` at the edge. **Keep the two in sync.**

## Added scope: web is now self-contained (`src/packages/*`)

Per a follow-up request, the 14 `@september/*` workspace packages were moved into `apps/web/src/packages/*` and de-workspaced:
- `@september/*` import specifier is **unchanged across all ~440 sites** — only the alias was repointed in `apps/web/tsconfig.json` to `./src/packages/*`.
- The 3 non-1:1 subpath exports needed explicit alias entries (before the glob): `@september/shared/{autocomplete,indexeddb,slides}` → their `lib/.../index.ts`.
- Each package keeps its `package.json` (for `main`/`exports` directory resolution); they are no longer pnpm workspace members. `pnpm-workspace.yaml` now globs only `apps/*`.
- Packages' external deps folded into `apps/web/package.json` (`@openrouter/ai-sdk-provider`, `@chenglou/pretext`, `diff-match-patch`, `react-resizable-panels`, `@types/diff-match-patch`).
- `vite.config.ts` keeps `ssr: { noExternal: [/^@september\//] }` so the raw-TS modules are transformed for the prerender pass.

## Other decisions

- **API routes dropped** (4 dormant handlers + `services/{gemini,elevenlabs}.ts`) — confirmed zero callers. Reintroduce as `createServerFn` if a server feature is ever needed.
- **`data/` → `src/data/`** (referenced via `@/data`; `@/*` now maps to `./src/*`).
- **Fonts**: `next/font/google` → self-hosted `@fontsource/noto-sans` (COEP-safe). Umami script kept with `crossOrigin: 'anonymous'`.
- **ESLint**: replaced `eslint-config-next` with a flat config tuned to the prior strictness (classic `react-hooks` rules, lenient TS) so the migration doesn't newly fail lint on untouched code. One safe `let`→`const` auto-fix applied. ~18 pre-existing warnings remain.
- **Tests**: 28 files / 347 tests pass. Added `apps/web/vitest.config.ts` (jsdom + `vite-tsconfig-paths` for alias parity); root `test` delegates to `--filter @september/web`; removed the now-unused root `vitest.config.ts`.

## Deployment — still needs a real preview deploy (plan step 9)

`vercel.json` sets the build command + COOP/COEP headers. Build output is a static SPA: `dist/client/` with prerendered `*/index.html` and `_shell.html` as the client-route fallback. **Not yet verified on Vercel** — confirm the SPA fallback (serve `_shell.html` for unmatched routes) and that headers land on the HTML document in a preview deploy before relying on prod.
