# @september/suggestions

AI-powered typing suggestions for the September composer.

## Public API

```ts
import { Suggestions, SuggestionsForm } from '@september/suggestions';
import type { SuggestionsFormData } from '@september/suggestions';
```

### `Suggestions`

Inline suggestion strip rendered below the composer. Shows two suggestion sources side-by-side:

1. **AI next-utterance suggestions** — generated by `useSuggestions` from the current chat history and user persona. Refetches on every new message; stale in-flight responses are dropped.
2. **Message-history matches** — past messages that match the current editor text (full-text search via `useMessages`).

```tsx
<Suggestions chatId={chatId} className="mt-2" />
```

Props: `chatId?: string`, `className?: string`.

### `SuggestionsForm`

Settings form shared by the settings page and the onboarding step. Renders all configuration fields (enable toggle, provider, model, system instructions, content corpus) and calls `onSubmit` with the validated data.

**Render-prop contract** — if you pass `children`, the form renders the fields then delegates submit chrome to the caller:

```tsx
<SuggestionsForm account={account} onSubmit={handleSubmit}>
  {({ form, hasApiKey, error, success }) => (
    <div className="flex justify-end gap-2">
      {error && <p className="text-red-500">{error}</p>}
      {success && <p className="text-green-600">Saved!</p>}
      <Button type="submit" disabled={form.formState.isSubmitting}>
        Save
      </Button>
    </div>
  )}
</SuggestionsForm>
```

If `children` is omitted the form renders its own submit button, error alert, and success indicator.

Props:
- `account?: Account` — current user account (pre-fills form defaults from `account.ai_suggestions`)
- `onSubmit: (data: SuggestionsFormData) => Promise<void>` — called with validated form data
- `children?: (props: { form, hasApiKey, error, success }) => React.ReactNode` — optional render prop for caller-supplied submit chrome

### `SuggestionsFormData`

TypeScript type inferred from the Zod schema. Shape:

```ts
{
  enabled: boolean;
  provider: 'gemini' | 'webllm';
  model: string;
  settings?: {
    system_instructions?: string;
    temperature?: number;
    max_suggestions?: number;
    context_window?: number;
    ai_corpus?: string;
  };
}
```

## Internals (not exported)

- `useSuggestions` — fetches AI suggestions via `useGenerate`; keyed on last message id
- `useCorpus` — generates a synthetic corpus from a persona description
- `Suggestion` interface — `{ text: string }`
- `SuggestionsFormSchema` — Zod schema backing `SuggestionsFormData`
